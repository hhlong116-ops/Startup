# Newborn/Baby Product Research Prototype

This prototype processes local CSV exports (no live scraping) to find newborn/baby products mentioned in social posts and visualize engagement + price data in Streamlit.

## Folder structure
```
.
├── app.py                  # Streamlit dashboard
├── data_pipeline.py        # ETL + aggregation from CSVs
├── utils_text.py           # Text cleaning + similarity helpers
├── aggregated_products.csv # Output generated by the pipeline (not committed)
├── social_posts.csv        # Input: social media posts export (local)
├── products_catalog.csv    # Input: product catalog / pricing feed (local)
├── image_matches.csv       # Optional: external image-to-product matches (local)
```

## Expected input schemas
- **social_posts.csv**
  - `post_id`: string
  - `image_id`: string or null
  - `image_url`: string or null
  - `caption`: string
  - `hashtags`: string
  - `likes`: int
  - `comments`: int
  - `posted_at`: datetime (string)
  - `platform`: string

- **products_catalog.csv**
  - `product_id`: string
  - `product_name`: string
  - `brand`: string
  - `model`: string or null
  - `category`: string
  - `price`: float
  - `currency`: string
  - `url`: string (e-commerce product URL)
  - `rating`: float or null
  - `marketplace`: string

- **image_matches.csv** (optional)
  - `image_id`: string
  - `product_id`: string
  - `score`: float (confidence)

## How to run
1. Place the CSV exports in the project root (matching the filenames above).
2. Generate aggregated metrics:
   ```bash
   python data_pipeline.py
   ```
3. Launch the dashboard:
   ```bash
   streamlit run app.py
   ```

## Notes
- All processing is based on local files; no scraping or unofficial API access is used.
- Keyword lists and matching thresholds live in `data_pipeline.py` and can be adjusted as needed.
